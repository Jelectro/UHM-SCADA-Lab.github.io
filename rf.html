Legacy Equipment

MDS entraNET 900 System Entranet Serial Remote

When Starting in fall 2022 the SCADA Lab was using the above MDS extender for radio tranmissions. The main problem that was happening was the connection was unstable. It would lose connection and not be able to re establish the connection on its own. To re-establish the connection we tried unplugging and re-plugging in the power, ethernet, and antenna cables on the control side MDS which did nothing. We repeated this on the production side and power cylcing work to re-establish the connection. So it was concluded the problem is on the production side MDS box. We were able to induce the problem by pinging the control side switch (10.1.1.10) from Qtpi-3(10.1.1.132) with a large packet using the command: ping 10.1.1.10 -s 10000.

In wireshark we saw the ARP table query: who has 10.1.1.10 tell 10.1.1.132. After about a second of no response QTpi-3 would try again: who has 10.1.1.10 tell 10.1.1.132. After around 3 Queries the OS of QTpi-3 would send the message Destination Host Unreachable.
